그보다 클땐 GNB의 모든 항목이 펼쳐보이게 만들었고, 그보다 작을 땐 햄버거 아이콘에 해당 항목들이 숨겨지게 제작했습니다.
그리고 햄버거 아이콘을 눌렀을 시 아래 사진 처럼 `Drawer` 컴포넌트가 왼쪽에서 열릴 수 있도록 제작했죠.

오늘은 해당 `Drawer` 컴포넌트에 관련된 얘기를 해보려 합니다.
해당 `Drawer`컴포넌트는 UI상으로나 애니메이션이나 딱히 문제는 없었습니다.
하지만 하나 걸리는 부분이 있었습니다. 바로 해당 컴포넌트의 구조상의 문제였습니다.

# 구조상의 문제?

저 `Drawer` 컴포넌트는 사실 의미론적으로나 HTML 구조적인 관점에서 봤을 때 전체 페이지를 덮고 있는 **전체 페이지에 대한 오버레이**라고 할 수 있습니다. (모달 같은 것도 마찬가지겠죠.)
그래서 다른 HTML요소 모든 것들의 위에 있어야 하는 것이죠.

하지만 제작한 `Drawer`는 현재 Next.js에서 기본적으로 생성하는 id가 \_\_next인 div내에 중첩되어 있습니다.

```js
<html>
  <head>...</head>
  <body>
    <div id="__next">
      <Drawer />
      ...
    </div>
  </body>
  ...
</html>
```

위에 작성한 구조가 현재 구조입니다.

하지만, 앞서 말했듯이 `Drawer` 컴포넌트는 사실 id가 \_\_next인 div와 같은 계층에 있어야 의미론적으로, 구조적으로 올바릅니다. 재차 말하지만, 전체 페이지에 대한 오버레이이기 때문이죠. 아래 코드처럼 말이죠.

```js
<html>
  <head>...</head>
  <body>
    <Drawer />
    <div id="__next">...</div>
  </body>
  ...
</html>
```

# 어떻게 해결할까?

해결해야하는 부분은 명확해졌습니다. 만든 해당 컴포넌트를 위와 같이 올바른 구조를 형성하는 위치로 이동시키는 것이죠.

하지만 컴포넌트 트리를 구성하는 일반적인 방법으로는 해결이 불가능합니다.
왜냐면 리액트 프로젝트 내에 컴포넌트들을 작성하면 그것들을 모두 커다란 하나의 커다란 div에 감싸서 반환하기 때문이죠.

## React의 Portals

React에서 제공하는 Portals는 이런 경우 좋은 해결법이 될 수 있습니다.
Portals는 기본적으로 상하 관계 구조를 가지고 있는 DOM에서 자식 컴포넌트를 부모 컴포넌트 밖에 있는 다른 컴포넌트로 전달할 수 있습니다.
사용법은 아주 간단합니다.

```js
ReactDOM.createPortal(child, container);
```

첫번째 인자인 child에는 전달하고자 하는 리액트 컴포넌트를 정의하면 되고,
두번째 인자인 container에는 첫번째 인자에 작성한 child를 보낼 DOM 요소를 정의하면 됩니다.

즉, container안에 새로운 div를 생성하지 않고 child 요소를 직계자식으로 렌더링하게 됩니다.

## CRA에서 적용

CRA로 만든 리액트 프로젝트에서는 기본적으로 `index.html` 파일을 제공합니다. 그곳에는 기본적으로 id가 root인 div요소가 있고 이 root DOM요소에서 모든 react 엘리먼트를 관리하죠. 하지만 우리는 이 root 요소에`Drawer`를 중첩하고 싶은 것이 아니고 같은 계층으로 배치해야합니다.

그러기 위해서는 먼저 `index.html`에 root 요소와 같은 계층으로 하나의 div를 작성하고 원하는 id를 부여하기만하면됩니다. 아래처럼요.

```js
[CRA] index.html

<html lang="en">
  <body>
    <div id="drawer"></div>
    // drawer를 보낼 컨테이너 정의
    <div id="root"></div>
    ...
  </body>
</html>
```

그 다음은 ReactDOM에서 제공하는 createPortal 메소드를 사용하여 우리가 옮기고 싶은 컴포넌트를 리턴하면 됩니다.

```js
[CRA] <Drawer />

const DrawerPortal = () => {
  const element = document.querySelector("#drawer")

  return ReactDOM.createPortal(<Drawer />, element)
}
```

물론 이렇게 꼭 `index.html`에 새로운 div를 적용하지 않고도 바로 원하는 컴포넌트를 body의 직계 자식으로 넣을 수도 있습니다.

```js
[CRA] <Drawer />

const DrawerPortal = () => {
  return ReactDOM.createPortal(<Drawer />, document.body)
}
```

이처럼 createPortal의 두번째 인자로 바로 document.body를 넣으면 됩니다. 하지만 id를 사용하는 것이 관리 측면에서 조금 더 적절해 보입니다.

## Nexst.js에서 적용

### 첫번째 문제 : document is undefined

기본적인 사용법은 같습니다. 하지만 Next.js에서는 기본적으로 처음에는 브라우저가 아닌 node 환경에서 프로젝트 내에 있는 코드를 실행하기 때문에 document객체가 뭔지 알 수가 없습니다.

document 객체는 클라이언트 사이드에서 컴포넌트가 마운트된 이후에 유효한 것이죠.
그래서 CRA와 완전히 같은 방법으로 접근했을 시에는 document is undefined라는 에러를 마주했습니다.

그래서 이것을 해결하기 위해 `<Portal />` 이란 래핑 컴포넌트를 하나 만들었습니다.

```js
// <Portal /> 컴포넌트

import { ReactNode, useEffect, useState } from "react";
import ReactDOM from "react-dom";

interface PortalProps {
  children: ReactNode;
  selector?: string;
}

const Portal = ({ children, selector }: PortalProps) => {
  const [element, setElement] = (useState < HTMLElement) | (null > null);

  useEffect(() => {
    setElement(document.getElementById(selector));
  }, [selector]);

  if (!element) return null;

  return ReactDOM.createPortal(children, element);
};

export default Portal;
```

이 해결법은 Next.js 공식문서에서도 소개가 되어있습니다. [공식문서 링크](https://nextjs.org/docs/messages/react-hydration-error)
서버에서 document 객체에 접근하지 못하게 하고, useEffect를 활용해 컴포넌트가 마운트 된 이후인 클라이언트 사이드에서 해당 document객체를 사용하는 방식으로 해결한 것입니다.
해당 컴포넌트의 기능을 요약하면 `<Portal />`은 기본적으로 element 상태를 가지고 있는데 기본값은 `null` 입니다. 그리고 이 상태가 null인 상태에서는 어떤 jsx도 리턴하지 않습니다.

해당 element 상태는 **컴포넌트가 마운트 된 이후에 비로소(클라이언트 사이드에서)** document 객체에 접근하여 props로 넘겨받은 selector를 통해 특정 DOM을 지칭하게 되고, 그 element를 createPortal의 두번째 인자로 넘겨 정상적으로 활용할 수 있게 됩니다.

마지막으로 포탈을 사용할 컴포넌트를 래핑하기만 하면 됩니다. 아래처럼요.

```js
// <Portal /> 컴포넌트

const App = () => {
  ...
  return
    (<>
     ...
     <Portal selector="drawer">
        <Drawer />
     </Portal>
     ...
    </>)
};

export default Portal;
```

여기까진 좋은데 Next.js 프로젝트에는 CRA와 다르게 html파일이 포함되어 있지 않습니다. 그러면 id가 drawer인 DOM요소는 어떻게 생성해야할까요?

### 두번째 문제 : Next.js에는 html 파일이 없다.

이는 Portal과 별개로 Next.js에서 제공하는 특별한 기능을 이해할 필요가 있는데 간단하게만 짚고 넘어가겠습니다.

### Next.js의 \_document.jsx(tsx)

우선 next 프로젝트에 \_document.jsx(tsx) 파일을 하나 생성하면 next.js에서는 그 파일을 해당 프로젝트의 기본적인 html 뼈대로 인식하게 됩니다. 해당 파일에는 **Next.js에서 제공하는** `<Head />`, `<Main />`, `<NextScript />` 컴포넌트가 필수적으로 필요합니다.
그것들을 아래처럼 정의하면 됩니다.

```js
import { Html, Main, Head, NextScript } from "next/document";

const Document = () => {
  return (
    <Html>
      <Head />
      <body>
        <Main />
        <NextScript />
      </body>
    </Html>
  );
};

export default Document;
```

이렇게 작성하면 Next.js는 이 구성을 우리 프로젝트 전체의 기본적인 HTML 뼈대로 인식하게 됩니다. 여기서 `<Main />` 컴포넌트에는 우리가 프로젝트에 작성하는 컴포넌트들이 포함되게 됩니다.

주의할 점은 이 Document 컴포넌트는 서버에서 한번만 실행되기 때문에 브라우저 관련한 코드를 작성할 수 없습니다.

아무튼 중요한 점은 이 특별한 기능(컴포넌트)을 통해 기본적인 html 뼈대를 커스터마이징 할 수 있다는 점입니다.
우리에게 지금 필요한 것은 id가 drawer인 div요소가 `<Main />` 컴포넌트와 같은 계층으로 존재하는 것입니다.
방법은 간단합니다. 해당 요소를 간단하게 Document 컴포넌트에 추가해주기만 하면 됩니다.

```js
import { Html, Main, Head, NextScript } from "next/document";

const Document = () => {
  return (
    <Html>
      <Head />
      <body>
        <div id="drawer"></div> // 추가
        <Main />
        <NextScript />
      </body>
    </Html>
  );
};

export default Document;
```

이렇게 하면 비로소 id가 drawer인 div요소가 생기고, 그 div요소에 실제로 제작한 `<Drawer />` 컴포넌트를 전달할 수 있게 되었습니다.

아래는 실제 제가 Next.js로 만든 블로그에 위의 내용들을 적용한 이후의 모습입니다. 사진 우측 엘리먼트 탭을 보시면 body의 자식으로 id가 \_\_next인 div요소와 drawer인 div요소가 형제요소로 존재하는 것을 확인하실 수 있습니다.

# 마무리

이렇게 React의 Portal을 소개해드렸습니다. 마지막 참고사항으로, 이렇게 React의 Portal은 DOM 트리의 어디든 존재할 수 있지만, 모든 다른 면에서 React 자식처럼 동작한다고 React의 공식문서에 소개가 되어있습니다.
Portal의 더 자세한 내용은 공식문서를 참고하시면 좋을 것 같습니다.
[리액트 공식문서-Portals링크](https://ko.reactjs.org/docs/portals.html)
